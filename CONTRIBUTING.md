- Here are some rules that we all have to follow for building this project so every one at same stage

## Branching

- Never commit on `main` branch, always create new branch for work
- use prefix

- `feature/feature-name`
- `fix/bug-fix-name`

## Commit message

- use words like
- `feat` - for feature implementation
- `fix` - for fixing bug
- `chore` - for changes that won't affect the `src` code files
- `test` - when test added or modified
- `revert` - when revert to previous commit

## Local Setup

- always pull latest changes from main branch

## Frontend

- `bun install` command for installing dependencies in `/frontend` folder
